About

Emacs Keys was created by Susam Pal with the intention of providing beginners to Emacs yet another way of learning Emacs. This page consists of a sequence of Emacs keys and commands that are carefully sequenced so that beginners to Emacs can find the most essential keys and commands very early on this page.

There is a demo in the form of screencast for each entry on this page. The length of each screencast does not exceed 10 seconds, so you can pick any entry on this page and you should be able to learn something useful about Emacs in less than a minute.

This website documents only the keys and commands available in vanilla GNU Emacs. Emacs distributions like Spacemacs, Doom Emacs, etc. are not covered here.

Frequenty Asked Questions (FAQ)

  1. How can I install Emacs?

    Visit GNU Emacs Download & Installation for more details.

  2. How can I make my Emacs look like the one in the screencasts?

    See this init.el that is used to customize Emacs to get the user interface you see in the screencasts. First copy the content of this file. Then start Emacs and type C-x C-f ~/.emacs.d/init.el RET. Then type C-y to paste. Then type C-x C-s to save. Finally type C-x C-c to quit. Start Emacs again and the user interface should look similar to the ones in the screencasts.

  3. Why can I not hear anything in the screencasts?

    There is no audio in the screencasts. All screencasts are silent videos.

  4. What tools do you use to make the screencasts?

    I use FFmpeg, the Swiss Army Knife for audio/video manipulation. I run a recent version of GNU Emacs with a tiny initialization file that makes Emacs look minimal and loads a dark color theme for it. FFmpeg is used to create a video capture of the screen while I demo the Emacs operations. The key sequence overlay that appears in green at the bottom of each video are subtitles added using FFmpeg. See the initialization file init.el, and the scripts record and encode for more details.

  5. How can I receive updates for new entries on this page?

    Go to the Subscribe section and subscribe to one or more channels mentioned there.

  6. Where can I report an issue I found on this page?

    You can report an issue or error you have found on this page at github.com/susam/emacskeys/issues. Click on the "New issue" button there to create a new issue. This is the preferred way of reporting an issue. If, however, you are not comfortable reporting an issue via GitHub, you can also talk to me via one of the channels mentioned in the Subscribe section. If you are familiar with Git and GitHub, you are also very welcome to send a pull request to fix issues.

  7. Where can I make a suggestion for adding a new entry to this page?

    See the answer to the previous question. You can use the same channels for suggesting a new entry for an Emacs key or command that is currently missing from this page.

  8. How can I support this page?

    If you have found this page useful, you can support this page in a couple of ways. You can send a donation via BMC. You can also subscribe to one or more of the channels mentioned in the Subscribe section, receive updates about new entries and share the links to the new entries with others.
